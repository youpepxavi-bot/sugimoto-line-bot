## 6. セグメント配信設計（将来）

`database.py` の `get_user_segment()` に基づくセグメント定義：

| セグメント | 条件 | 配信内容 |
|-----------|------|--------|
| `new` | 来店1回 | 次回来店促進クーポン |
| `regular` | 来店5〜19回 | VIPへの誘い、新メニュー情報 |
| `vip` | 来店20回以上 | 先行情報、VIP限定特典 |
| `lapsed` | 最終来店から60日以上 | 再来店感謝クーポン |

### セグメント配信実装例（将来コード）
```python
# main.py に追加予定
@app.post("/admin/segment-broadcast")
async def segment_broadcast(request: Request):
    data = await request.json()
    segment = data["segment"]  # "regular", "vip", "lapsed"

    db = SessionLocal()
    # lapsed: 最終来店から60日以上
    from datetime import date, timedelta
    cutoff = date.today() - timedelta(days=60)
    users = db.query(User).filter(
        User.last_visit_date < cutoff,
        User.is_following == True
    ).all()

    # Multicast送信（最大500件/回）
    user_ids = [u.line_user_id for u in users]
    # ... 500件ずつ分割してMulticastRequestで送信
```
